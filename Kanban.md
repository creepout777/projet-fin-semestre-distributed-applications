# Sprint 1
- [x] Account Registration: As an Admin, I want to register users with specific roles (Admin, Scolarité, Enseignant, Etudiant) in order to provide authorized access to the campus system.

- [x] User Authentication: As a User, I want to log in securely with my credentials in order to access the features and data restricted to my specific role.

- [x] Session Management: As a User, I want to maintain my login session securely for a fixed duration in order to avoid frequent re-authentication.

- [ ] Password Recovery: As a User, I want to request a password reset via email so that I can regain access if I forget my credentials.

- [x] Role-Based Access Control (RBAC): As an Admin, I want the system to enforce permissions at the endpoint level based on the user's assigned role in order to protect sensitive modules.






# Sprint 2

- [ ] Create Slot: As an Admin or Scolarité, I want to create a schedule slot (créneau) with date and time in order to plan academic sessions.

- [ ] Update Slot: As an Admin or Scolarité, I want to modify an existing slot in order to adjust scheduling changes.

- [ ] Delete Slot: As an Admin or Scolarité, I want to delete a slot in order to remove incorrect or canceled sessions.

- [ ] Assign Slot Details: As an Admin or Scolarité, I want to associate a slot with a module, group, teacher, and room in order to fully define a session.

- [ ] Validate Slot Conflicts: As a System, I want to detect scheduling conflicts (teacher, room, group) in order to prevent overlapping sessions.

- [ ] Daily View: As a User, I want to view the schedule by day in order to see sessions planned for a specific date.

- [ ] Weekly View: As a User, I want to view the schedule by week in order to have a broader planning overview.

- [ ] Filter by Student/Group/Teacher: As a User, I want to filter the schedule by student, group, or teacher in order to see relevant sessions only.
